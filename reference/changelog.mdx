<AccordionGroup>

<Accordion title="2.x releases">

**2.4.0** — 2023-09-22
- Add Citus 12 to the test matrix.
- Add support for Rails 7.1.
- Fix missing scope in `habtm.rb`.
- Update logic inside the `tenant_klass_defined?` method.

**2.3.0** — 2023-06-05
- Add `has_and_belongs_to_many` feature with tenant support.
- Remove end-of-life Ruby versions from the test matrix.
- Add documentation on the ReadTheDocs platform.
- Organize badges in documentation and `README.md`.
- Wrap `ActiveRecord::Base` with `ActiveSupport.on_load`.

**2.2.0** — 2022-12-06
- Handle changing tenant from `nil` to a value.
- Allow partitioned tables to be created without a primary key.
- Only attempt to reload with `MultiTenant` when `partition_key` is present.
- Remove support for Ruby 2.5 and ActiveRecord 5.2.

**2.1.6** — 2022-11-23
- Fix undefined `wrap_methods` error and `wrap_methods` version check.

**2.1.5** — 2022-11-20
- Fix `MultiTenant.without` code-generation bug in Rails 6.1+.

**2.1.4** — 2022-11-03
- Fix `db:schema:dump` breakage when using this gem with MySQL.

**2.1.3** — 2022-10-27
- Fix error when calling a method that takes keyword arguments via `MultiTenant.wrap_methods`.

**2.1.2** — 2022-10-26
- Fix issue when wrapping methods that require a block.

**2.1.1** — 2022-10-20
- Fix query building for models with mismatched `partition_key`s.
- Identify tenant even if the class name is non-standard.
- Add `current_tenant_id` to `WHERE` clauses when calling methods on an ActiveRecord instance or its associations.
- Make `create_distributed_table` and `create_reference_table` reversible; add Ruby wrapper for `rebalance_table_shards`.
- Support `create_distributed_table` and `create_reference_table` in `schema.rb`.
- Add client and server Sidekiq middleware to the Sidekiq middleware chain automatically.

**2.0.0** — 2022-05-19
- Replace `RequestStore` with `ActiveSupport::CurrentAttributes` for tenant storage.
- Support changing `table_name` after calling `multi_tenant`.
- Allow UUID as the primary key on partition tables.
- Support latest Rails 5.2.
- Support `optional: true` for the auto-generated `belongs_to`.

</Accordion>

<Accordion title="1.x releases">

**1.2.0** — 2022-03-29
- Test Rails 7 and Ruby 3.
- Fix regression in 1.1.1 involving deleted tenants.
- Fix incorrect SQL generated when joining two models and one has a default scope.
- Update for Rails 5+ removal of `type_cast_for_database`.

**1.1.1** — 2021-01-15
- Add support for Rails 6.1.
- Fix statement cache for `has_many :through` relations.

**1.1.0** — 2020-08-06
- Various fixes and improvements. See the [commit history for v1.1.0](https://github.com/citusdata/activerecord-multi-tenant/commits/v1.1.0).

**1.0.4** — 2019-10-30
- Fix bug introduced in 1.0.3 for `delete` when the table is a reference table or not distributed.

**1.0.3** — 2019-10-28
- Ensure that `object.delete` sets the tenant before executing.

**1.0.2** — 2019-09-20
- Compatibility with Rails 6.
- Remove support for Rails 4.0 and 4.1.
- Fix bug when multiple databases are used.

**1.0.1** — 2019-08-27
- Ensure the current tenant is present before adding the tenant ID filter in `DatabaseStatements`.

**1.0.0** — 2019-07-05
- Fix `RETURNING id` for distributed tables with no primary key.
- Fix `ActiveModel::MissingAttributeError` raised when doing a partial `select` without the tenant column (e.g. `Project.select(:name).find(id)`).

</Accordion>

<Accordion title="0.x releases">

**0.11.0** — 2019-06-12
- Fix queries with joins by including the tenant column when the current tenant is not set. A common use case is filtering on the tenant without `MultiTenant.with`, e.g. `Project.where(account_id: 1).eager_load(:categories)`.

**0.10.0** — 2019-05-31
- Add `MultiTenant.without` to remove an already-set tenant context inside a block.
- Fix uninitialized constant `X::ActiveRecord::VERSION`.
- Fix `find` and `find_by` caching issues — tenant ID was being cached across different tenant contexts when prepared statements were enabled.
- Add `MultiTenant.load_current_tenant!` to load the full tenant object when only an ID is set.
- Update `loofah` and `rack` to address security warnings in the test suite.
- Remove the monkey patch that previously disabled referential integrity (`DISABLE/ENABLE TRIGGER ALL`) — the underlying Citus issue has been resolved upstream.

**0.9.0** — 2018-06-22
- Add ActiveRecord 5.2 support.

**0.8.1** — 2017-10-06
- Cast attribute name to a string to avoid double-applying the tenant clause.
- Allow bulk `DELETE`/`UPDATE` with subqueries on joins.

**0.8.0** — 2017-08-16
- Significant improvements and simplifications to query rewriting. Fixes caching issues across multiple `MultiTenant.with {}` blocks when interacting with the Rails statement cache.
- Drop support for Rails 3.2.

**0.7.0** — 2017-07-18
- Switch back to a `Relation`-based mechanism for hooking into ActiveRecord (resolves issues for simple queries that were not being rewritten).
- Handle `OUTER JOIN` correctly in the query rewriter.
- Correctly rewrite sub-selects.
- Only return the cached tenant object if the association is not already loaded.
- Fix support for inherited model classes that only have `multi_tenant` on a higher-level object.
- Sidekiq middleware: do not automatically perform a `find` for the tenant object for every job.
- Fix automatic `inverse_of` on singular associations.
- Fix bug that prevented `FastTruncate` from running.

**0.6.0** — 2017-06-09
- Change the query rewriter hook from per-relation to pre-SQL-statement output — resolves cases where conditions were added in the wrong place.
- Use a table name to model class registry inside the query rewriter.
- Improve tests: use a lower shard count to speed up the suite; drop the DatabaseCleaner dependency.

**0.5.0** — 2017-05-08
- Introduce write-only mode for step-by-step migrations.
- Add tenant ID to queries using a rewriter instead of `default_scope`/`unscoped`.
- Add query monitor that warns about missing tenant IDs.
- Add `FastTruncate` helper.
- Add Sidekiq middleware.

**0.4.1** — 2017-03-23
- Allow use outside of Rails (e.g. Sinatra with ActiveRecord).

**0.4.0** — 2017-03-22
- Infer `multi_tenant` setting from parent classes.
- Remove use of the global tenant class variable.
- Support passing raw ID values to `MultiTenant.with` directly (deprecates `with_id`, which remains as an alias).
- Remove unnecessary validation for an invalid `belongs_to` association.

**0.3.4** — 2017-02-22
- Expand the `with_lock` workaround to cover `lock!` as well.
- Enable the trigger workaround on Rails 5 (it was silently failing).
- Switch tests to the DatabaseCleaner truncation strategy to avoid multi-shard transactions.

**0.3.3** — 2017-02-21
- Avoid warning about multi-column primary keys with Rails 5.
- Fix odd bind errors for `has_one`/`has_many :through`.
- Add `MultiTenant.current_tenant_id=` helper method.

**0.3.2** — 2017-02-16
- Support blocks passed into the `unscoped` method (fixes `reload`, among other issues).
- Make `with_lock` work by adding a workaround for Citus issue #1236 (`SELECT ... FOR UPDATE` was not router-plannable).

**0.3.1** — 2017-02-13
- Rails 5 API controller support.
- Citus 6.1 compatibility.

**0.3.0** — 2016-12-30
- Remove dependency on `acts_as_tenant`; copy only the necessary code.
- Fix issue with callbacks receiving a `TenantIdWrapper` instead of the actual object.

**0.2.1** — 2016-12-29
- Fix bug in the `CopyFromClient` helper.

**0.2.0** — 2016-12-27
- Initial release.

</Accordion>
